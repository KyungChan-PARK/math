스마트 도시의 교통 시스템

🌆 상황 설명

서울시는 새로운 스마트 교통 시스템을 도입했습니다. 이 시스템은 다음과 같은 데이터를 수집합니다:

• 시스템 A: 각 차량의 번호판을 읽어 해당 차량의 현재 속도를 기록
• 시스템 B: 특정 속도로 주행 중인 모든 차량의 번호판을 기록
• 시스템 C: 운전자의 휴대폰 번호로 차량 위치를 추적
• 시스템 D: 특정 지역에 있는 모든 운전자의 휴대폰 번호를 수집

교통 관제 센터에서는 "하나의 입력에 대해 정확히 하나의 출력을 보장하는 시스템"만을 신뢰할 수 있는 데이터 소스로 분류합니다.
위 시스템 중 신뢰할 수 있는 데이터 소스로 분류될 수 있는 것은 무엇이며, 그 이유를 수학적 관계를 통해 설명하시오.

📖 상세 설명과 사고 과정

1단계: 문제의 핵심 파악
"하나의 입력에 정확히 하나의 출력" = 함수의 정의
우리가 찾아야 할 것: 어떤 시스템이 함수 관계를 만족하는가?
여기서 잠깐! 왜 "하나의 입력에 하나의 출력"이 신뢰성과 연결될까요?
같은 번호판을 입력했는데 매번 다른 속도가 나온다면, 그 데이터를 믿을 수 있을까요?
시스템 A: 번호판 → 속도

✓ 함수 (각 차량은 한 순간에 하나의 속도)

시스템 B: 속도 → 번호판들

✗ 함수 아님 (여러 차량이 같은 속도)

2단계: 각 시스템 분석

시스템 A (번호판 → 속도):
• 12가3456 → 60km/h
• 한 차량이 동시에 60km/h이면서 80km/h일 수 있나? ❌
• 결론: 함수입니다 ✓

시스템 B (속도 → 번호판):
• 60km/h → {12가3456, 78다9012, 56라3456, ...}
• 여러 차량이 같은 속도로 달릴 수 있나? ⭕
• 결론: 함수가 아닙니다 ✗
깨달음의 순간: 시스템 A와 B의 차이점
A는 "이 차의 속도는?" (답은 하나)
B는 "60km/h로 달리는 차는?" (답이 여러 개)
방향이 바뀌면 함수 관계도 바뀝니다!

• 결론: 함수입니다 ✓

온라인 쇼핑몰의 재고 관리 시스템

📦 상황 설명

대형 온라인 쇼핑몰에서 두 가지 재고 관리 시스템을 테스트하고 있습니다.

드론 배송 경로 최적화

🚁 상황 설명

한 물류 회사는 드론을 이용한 배송 서비스를 계획하고 있습니다. 드론의 비행 경로를 설계하기 위해 세 가지 경로 모델을 검토 중입니다:

모델 A: 직선 경로
높이(h) = 2 × 거리(d) + 10 (단위: 미터)

모델 B: 원형 경로
경로가 방정식 d² + h² = 100을 따름

모델 C: 포물선 경로
높이(h) = -0.1 × d² + 5 × d (단위: 미터)

안전 규정상 "각 수평 위치에서 드론의 높이가 정확히 하나로 결정되는 경로"만 승인됩니다.
세 가지 모델 중 안전 규정을 만족하는 것을 모두 찾고, 각 모델이 실제 비행에서 어떤 의미를 갖는지 설명하시오.

📖 상세 설명과 사고 과정

1단계: 안전 규정의 수학적 해석
"각 수평 위치(d)에서 높이(h)가 하나로 결정" = d를 입력으로 하는 함수!
이는 수직선 검사를 통과해야 한다는 의미입니다.
모델 A: 직선 경로
모델 B: 원형 경로
모델 C: 포물선 경로
물리적 직관과 수학의 만남
원형 경로에서 드론이 "되돌아온다"는 것은 같은 수평 위치에 두 번 도달한다는 의미!
이는 충돌 위험을 의미하며, 바로 이것이 함수가 아닌 경로가 위험한 이유입니다.
2단계: 각 모델의 수학적 검증

모델 A: h = 2d + 10
• d = 0 → h = 10 (유일)
• d = 10 → h = 30 (유일)
• d = 20 → h = 50 (유일)
각 거리에서 높이가 유일하게 결정 → 함수 ✓

모델 B: d² + h² = 100
• d = 0일 때: h² = 100 → h = ±10
• d = 6일 때: h² = 64 → h = ±8
같은 거리에서 두 가지 높이 가능 → 함수 아님 ✗

모델 C: h = -0.1d² + 5d
• d = 0 → h = 0 (유일)
• d = 25 → h = 62.5 (유일)
• d = 50 → h = 0 (유일)
각 거리에서 높이가 유일하게 결정 → 함수 ✓
🎯 실제 비행의 의미
모델 A
"상승 비행"
계속 올라감
→
모델 B
"순환 비행"
되돌아옴 (위험!)
→
모델 C
"포물선 비행"
올라갔다 내려옴
3단계: 실용적 의미

모델 A (직선)의 장점:
• 예측 가능하고 단순
• 연료 소비 일정
• 긴 거리 배송에 부적합 (너무 높이 올라감)

모델 B (원형)가 위험한 이유:
• 같은 위치를 두 번 지남
• 다른 드론과 충돌 위험
• 제어 시스템 혼란

모델 C (포물선)의 장점:
• 자연스러운 비행 경로
• 중간 지점에서 최고 높이
• 착륙 시 안전한 하강
최종 통찰: 함수의 실용성
"함수 = 예측 가능성 = 안전성"
항공, 교통, 통신 등 모든 시스템에서 함수 관계는 신뢰성의 기초입니다.
수학적 엄밀성이 실제 안전으로 이어지는 완벽한 예시!

스트리밍 서비스 요금 전략

🎬 상황 설명

새로운 동영상 스트리밍 서비스 "WatchMax"는 다음과 같은 복잡한 요금 체계를 도입했습니다:

• 기본 요금: 월 9,900원
• 추가 프로필당: 2,000원 (최대 4개까지)
• 4K 화질 옵션: 3,000원
• 동시 시청 기기 수에 따른 추가 요금:
- 2대까지: 추가 요금 없음
- 3-4대: +3,000원
- 5대 이상: +6,000원

마케팅팀은 "고객이 선택한 옵션에 따라 월 요금을 정확히 계산할 수 있는 시스템"을 요구합니다.
위 요금 체계를 수식으로 표현하고, 다양한 고객 시나리오에 대한 요금을 계산하시오. 또한 이 요금 체계가 함수로 표현 가능한지 분석하시오.

📖 상세 설명과 사고 과정

1단계: 변수 정의와 요금 구조 분석

입력 변수들:
• p: 추가 프로필 수 (0 ≤ p ≤ 4)
• q: 4K 화질 선택 (0 또는 1)
• d: 동시 시청 기기 수

출력:
• C: 총 월 요금
첫 번째 도전: 다변수 함수
지금까지는 하나의 입력(x)을 다뤘지만, 실제로는 여러 입력이 있을 수 있습니다.
C = f(p, q, d) 형태로, 세 개의 입력이 하나의 출력을 결정합니다!
2단계: 요금 함수 구성

기본 구조:
C = 9900 + (프로필 요금) + (4K 요금) + (기기 요금)

세부 계산:
C = 9900 + 2000p + 3000q + D(d)

여기서 D(d)는 기기 수에 따른 추가 요금:
D(d) = {
  0 (d ≤ 2)
  3000 (3 ≤ d ≤ 4)
  6000 (d ≥ 5)
}

시나리오 2: 프리미엄 가족

3단계: 함수성 검증

질문: 같은 선택 조합이 다른 요금을 낼 수 있는가?

테스트:
• (2, 1, 3) → 9900 + 4000 + 3000 + 3000 = 19,900원
• (2, 1, 3) 다시 입력 → 여전히 19,900원 ✓

• (0, 0, 2) → 9,900원
• (0, 0, 2) 다시 입력 → 여전히 9,900원 ✓

결론: 이것은 함수입니다!
각 (p, q, d) 조합은 정확히 하나의 요금 C를 결정합니다.
실무 통찰: 함수적 사고의 중요성
요금 체계가 함수가 아니라면?
• 같은 서비스에 다른 요금 → 고객 불만
• 자동 결제 시스템 구축 불가능
• 법적 문제 발생 가능

함수적 설계 = 공정성과 투명성의 보장!
4단계: 실제 구현 코드 (의사코드)

function 월요금계산(프로필수, 4K여부, 기기수) {
    기본요금 = 9900
    프로필요금 = 2000 × 프로필수
    화질요금 = 3000 × 4K여부

    if (기기수 ≤ 2) {
        기기요금 = 0
    } else if (기기수 ≤ 4) {
        기기요금 = 3000
    } else {
        기기요금 = 6000
    }

    return 기본요금 + 프로필요금 + 화질요금 + 기기요금
}
최종 깨달음: 현대 비즈니스와 함수
• 우버 요금 = f(거리, 시간, 수요)
• 신용점수 = f(연체기록, 소득, 부채비율)
• 추천 알고리즘 = f(시청기록, 선호도, 인기도)

함수는 디지털 경제의 언어입니다!

AI 챗봇의 응답 예측 모델

🤖 상황 설명

한 IT 회사가 고객 서비스용 AI 챗봇을 개발했습니다. 챗봇의 응답 시간을 예측하기 위해 다음 데이터를 수집했습니다:

• 질문 복잡도(x): 1-10점 척도
• 응답 시간(f(x)): 초 단위

개발팀은 세 가지 모델을 제안했습니다:
모델 α: f(x) = x² - 2x + 3
모델 β: f(x) = 2x + 1
모델 γ: f(x) = (x-1)² + 2

품질 관리팀은 "복잡도 0과 2에서 같은 응답 시간을 보이는 대칭적 모델"을 찾고 있습니다.

📖 상세 설명과 사고 과정

1단계: 함수 기호의 이해

f(x)는 "복잡도 x를 넣으면 응답 시간을 알려주는 기계"
• f: 함수 이름 (function)
• x: 입력 (질문 복잡도)
• f(x): 출력 (응답 시간)
사고의 전환: f(a+1)의 의미
"복잡도가 (현재 복잡도 + 1)일 때의 응답 시간"
이는 복잡도가 1 증가했을 때의 일반적인 패턴을 보여줍니다!

3단계: 실제 의미 해석

모델 α, γ (포물선)의 해석:
• 복잡도 1에서 최소 응답 시간 (2초)
• 너무 쉬운 질문(x=0)도 오래 걸림 → 맥락 파악 필요
• 너무 어려운 질문도 오래 걸림 → 처리 복잡도 증가
• 중간 난이도가 최적 → AI가 가장 잘 처리하는 영역

모델 β (직선)의 해석:
• 복잡도에 비례하여 시간 증가
• 단순하지만 비현실적
• 학습 효과나 최적화 없음

최종 통찰: 함수 변형의 의미
• f(x) = x² - 2x + 3 (전개형): 계산에 적합
• f(x) = (x-1)² + 2 (완성형): 그래프 특성 파악에 적합

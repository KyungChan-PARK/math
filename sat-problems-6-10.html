소셜 미디어 바이럴 패턴 분석

📱 상황 설명

한 마케팅 회사는 소셜 미디어 게시물의 확산 패턴을 분석하고 있습니다. 게시물의 도달률을 예측하기 위해 세 가지 모델을 테스트했습니다:

모델 A (선형 성장):
도달수 = 500 × 시간(시) + 1000

모델 B (지수 성장):
도달수 = 1000 × 2^(시간/3)

모델 C (로그 성장):
도달수 = 5000 × log₂(시간 + 1) + 1000

마케팅팀은 "처음 12시간 동안의 패턴"을 기반으로 광고 예산을 배정하려 합니다.
각 모델이 나타내는 확산 패턴의 특징을 분석하고, 어떤 유형의 콘텐츠에 각 모델이 적합한지 설명하시오. 또한 t=3, t=6, t=12에서의 도달수를 비교하시오.

• 선형 함수: 일정한 속도로 증가
• 지수 함수: 점점 빠르게 증가
• 로그 함수: 처음엔 빠르다가 점점 느려짐
시간 t=3

스마트폰 배터리 최적화 알고리즘

🔋 상황 설명

스마트폰 제조사는 배터리 수명을 늘리기 위해 조각 함수(piecewise function)를 사용한 전력 관리 시스템을 개발했습니다:

배터리 잔량(b%)에 따른 CPU 성능(p%):
p(b) = {
  100% (b ≥ 50)
  1.5b + 25 (20 ≤ b < 50)
  30% (b < 20)
}

사용자는 하루 동안 배터리가 100%에서 0%로 선형적으로 감소한다고 가정합니다.
하루 24시간 동안 CPU 성능의 변화를 그래프로 나타내고, 각 구간에서의 사용자 경험을 분석하시오. 또한 이 시스템의 장단점과 개선 방안을 제시하시오.

📖 상세 설명과 사고 과정

1단계: 시간에 따른 배터리 감소

배터리 = 100 - (100/24) × 시간
= 100 - 4.17 × 시간

주요 시점:
• 0시간: 100% 배터리
• 12시간: 50% 배터리 (경계점 1)
• 19.2시간: 20% 배터리 (경계점 2)
• 24시간: 0% 배터리

• 충분한 배터리: 제한 없는 성능
• 중간 배터리: 점진적 조절
• 낮은 배터리: 생존 모드
각 구간이 다른 전략을 구현
2단계: 각 구간의 의미와 사용자 경험

구간 1 (배터리 50-100%, 0-12시간):
• CPU 100% - 모든 기능 정상 작동
• 게임, 동영상, 멀티태스킹 가능
• 사용자 만족도: 높음

구간 2 (배터리 20-50%, 12-19.2시간):
• CPU 55-100% - 점진적 성능 저하
• 배경 앱 제한, 화면 밝기 감소
• 사용자 인지: "폰이 느려지네"

• 필수 기능만 작동
• 사용자 반응: "충전기 어디있지?"

음악 스트리밍 추천 알고리즘

🎵 상황 설명

음악 스트리밍 서비스 "MelodyAI"는 사용자의 청취 패턴을 분석하여 다음 곡을 추천하는 알고리즘을 개발했습니다:

유사도 점수 계산:
두 곡 A와 B의 유사도 = f(템포차이, 장르일치, 분위기)

템포 점수: T(x) = 100 - |x| (x = 템포A - 템포B)
장르 점수: G = {100 (같은 장르), 50 (유사 장르), 0 (다른 장르)}
분위기 점수: M = 20 × (5 - |분위기A - 분위기B|)

최종 유사도 = 0.4T + 0.3G + 0.3M
현재 재생 중인 곡(템포 120, 팝, 분위기 4)과 세 후보곡의 유사도를 계산하고, 이 알고리즘이 만들어내는 추천 패턴을 분석하시오. 또한 함수 합성의 관점에서 이 시스템을 해석하시오.

📖 상세 설명과 사고 과정

1단계: 후보곡 정보와 유사도 계산

현재곡: 템포 120, 팝, 분위기 4

후보 A: 템포 125, 팝, 분위기 4
• T(5) = 100 - 5 = 95
• G = 100 (같은 장르)
• M = 20 × (5 - 0) = 100
• 유사도 = 0.4(95) + 0.3(100) + 0.3(100) = 98

후보 B: 템포 100, 록, 분위기 5
• T(-20) = 100 - 20 = 80
• G = 50 (유사 장르)
• M = 20 × (5 - 1) = 80
• 유사도 = 0.4(80) + 0.3(50) + 0.3(80) = 71

후보 C: 템포 180, 일렉, 분위기 1
• T(60) = 100 - 60 = 40
• G = 0 (다른 장르)
• M = 20 × (5 - 3) = 40
• 유사도 = 0.4(40) + 0.3(0) + 0.3(40) = 28

자율주행차 안전거리 계산

🚗 상황 설명

자율주행 자동차는 속도에 따라 안전거리를 실시간으로 계산합니다. 세 가지 모델이 제안되었습니다:

안전거리 모델 (v: 속도 km/h, d: 거리 m):
• 모델 A (반응시간 중심): d = 0.5v + 10
• 모델 B (제동거리 중심): d = 0.01v² + 0.2v
• 모델 C (복합 모델): d = 0.005v² + 0.3v + 5

교통 안전 당국은 "도시(30-50km/h)와 고속도로(80-120km/h) 모두에서 적절한 안전거리를 보장하는 모델"을 요구합니다.
각 모델의 안전거리를 속도 30, 50, 80, 120km/h에서 계산하고, 실제 도로 상황에서의 적합성을 평가하시오. 또한 함수의 성장률이 안전에 미치는 영향을 분석하시오.
